## 二、LED 基礎（數位輸出）

### 變數定義的好處

```cpp
int led1 = 12;  // 用變數定義腳位
```

- 之後要換腳位，只改這一行，程式內部都不用動
- 這就是「變數」的意義：一處修改，全部生效

### setup() 與 loop()

- `setup()`：初始化，只跑一次。設定腳位方向（輸入/輸出）
- `loop()`：主迴圈，重複執行

### pinMode 方向

- `OUTPUT`：丟出去（LED、蜂鳴器）
- `INPUT`：抓進來（按鈕、感測器）

### 註解

- `//` 雙斜線 = 註解，給人看的，程式編譯時會忽略

## 三、呼吸燈（PWM）

### 重點觀念

- 呼吸燈需要漸變效果，不是單純的開/關
- **必須接在有 `~` 標示的 PWM 腳位**（如 9, 10, 11）
- `analogWrite(pin, 0~255)` 控制亮度

### 參數調整

- `delay` 值影響速度：
  - `delay(1)` = 0.001 秒 → 變化快
  - `delay(30)` = 0.03 秒 → 變化慢，更平順
- `fadeStep` 值影響每步變化量

## 六、蜂鳴器（tone）

### 有源 vs 無源蜂鳴器

| 類型 | 特性 | 用途 |
|------|------|------|
| **有源**（帶振盪源） | 通電就叫，固定頻率 | 門鈴、警報器、倒車提示 |
| **無源**（不帶振盪源） | 需要給頻率訊號才會響 | 播音樂、自訂音效 |

- 課堂用的是**無源蜂鳴器**，可以控制頻率來演奏旋律
- 頻率 = 音高：人耳聽覺範圍 20Hz ~ 20,000Hz（20kHz）

### 音階與頻率

- 每個音對應一個頻率（如 Do=262Hz, La=440Hz）
- 手機調音器 APP 就是比對聲音頻率是否符合音階
- `tone(pin, frequency)` 產生指定頻率的方波

### 接線

- 板子上蜂鳴器在 **D5**（預設範例可能寫 D8，要改）
- 上傳空程式可以讓蜂鳴器停止

## 七、光敏電阻（類比輸入）

### 觀念

- 光敏電阻偵測環境光的**強度**，有大小之分 → **類比訊號**
- 類比訊號接在 **A 開頭的腳位**（板子上是 A1）
- `analogRead()` 讀取 0~1023 的值

### 分壓電路

- 光敏電阻需要搭配一個 **10kΩ 電阻**組成分壓電路
- Arduino 只能量電壓，不能直接量電阻值
- 分壓電路把「阻值變化」轉成「電壓變化」

### Serial Monitor

- `Serial.begin(9600)` 設定鮑率為 9600
- 實體板子透過 USB 傳數值到電腦上顯示
- 模擬器也有類似功能，旁邊有個示波器圖示可以看圖形化變化

| 元件 | 板子腳位 | 範例預設 | 要改！ |

| LED2（紅） | **D12** | D3 | 要 |
| RGB LED | **D9/D10/D11** | - | - |
| Buzzer | **D5** | D8 | 要 |
| Light | **A1** | A0 | 要 |

## 九、實作注意事項

- 擴展板背面有金屬焊點，**不要放在金屬表面上**，會短路
- 板子上的元件都是焊死的，不需要麵包板接線
- 從模擬器搬程式碼到實體板子時，**腳位一定要改**
- 開發板選擇：工具 → 開發板 → AVR → Arduino UNO
